## 一些经验？

1.让AI问问题！（苏格拉底式提问法）

让AI自己想办法与我们对齐！并且哪怕在你有一定的想法之后，也不要先告诉AI，万一AI能找到更好的路径呢？
2.学生与专家解释法（费曼学习法？）

当我们遇到一个新的概念时，或者说一个新的代码库，我通常会让AI让我尽可能容易的理解。这需要两个输入，一是使用诸如举例、类比的方式，二是人为的降低我们自己的身份。

第一层prompt：

```
我是一名中国大陆的理科高中生，请你用尽可能同时易懂的语言，给我解释XXX概念/@项目，可以使用诸如举例说明、类比等尽可能形象的方式，使得我快速且容易地理解。
```

上述prompt可以完成快速入门，但是并不代表我们可以进行深挖，于是我们还需要专业的视角。

第二层prompt：

```
请你作为专家，用A领域的公式和专业性的语言对XXX进行描述，注意使用公认的字母且不能出现事实性错误
```

3.让质疑存在，让角度更全面！

哪怕用户所输入的信息和信息之间逻辑互斥，AI均会默认用户所输入的是所有信息都是对的，并且还会一本正经非常严肃地对互斥的逻辑进行解释&掩盖，语气给你感觉是非常值得信服的，事实上是在扯淡。

哪怕我们的输入是正确的，但对结果进行review的代价也很大。于是，我们需要加入质疑环节：

```
请你作为一个很mean的审稿人，对XXX回答进行批判性的思考，你需要基于事实和严谨的逻辑，去寻找漏洞和可能存在的风险。你不需要客气，你只需要严厉的攻击
```
